# 主题 Token 清单

本文档定义应用级主题 token。它覆盖整个应用，而不只是画布：Web 界面、shadcn 组件、Iconoir 图标、Konva 编辑画布、源码面板、Markdown 视图、终端和 Mermaid 渲染视图都必须从同一套 token 派生。

## 设计原则

- 用户可编辑的是语义 token，不直接编辑具体组件里的硬编码样式。
- shadcn CSS 变量、Konva visual token、Mermaid themeVariables 和 xterm theme 都是派生结果。
- 浮动控件属于应用 UI 控件，不是装饰层；它们必须复用按钮、图标、边框、阴影和圆角 token。
- 实线、虚线、节点形状、箭头等图语义不属于主题 token；它们是 Mermaid/图模型语义。但这些语义的视觉表现可以由 token 控制。
- 对象级样式覆盖优先级高于应用主题，但必须只覆盖有限字段，例如节点填充、描边、文字和连线样式。

## 优先级

1. 平台与渲染器基础默认值
2. 应用主题 token
3. 文件级主题配置
4. 对象级样式覆盖
5. 交互状态临时覆盖
6. 主题设置页预览覆盖

### Markdown

Markdown 阅读、编辑和浮动窗口共享 `theme.markdown`，并编译为 `--markdown-*` CSS 变量。当前结构按文档元素组织外观：一个元素的排版、颜色、布局和边框都放在同一对象内，不再单独维护 `markdown.typography`。内容宽度属于应用阅读偏好，不属于主题 token。

字段路径、控件类型、范围、单位、默认值及其主题派生来源统一定义在 Markdown Token 目录中；默认主题、归一化和设置面板都直接消费该目录。下面的矩阵是这份目录的阅读版，不再单独发明另一套分组。

#### 元素与通用文字字段

下表中的 `<element>` 可替换为：

`body`、`heading.h1`、`heading.h2`、`heading.h3`、`heading.h4`、`heading.h5`、`heading.h6`、`link`、`emphasis`、`strong`、`strikethrough`、`list.unordered`、`list.ordered`、`list.task`、`blockquote`、`inlineCode`、`codeBlock`、`table`。

这些元素都直接拥有下列六个字段；`divider` 和 `image` 没有文字字段。

| Token | 含义 | 类型 / 单位 | 范围 | 默认值 / 派生来源 |
| --- | --- | --- | --- | --- |
| `markdown.<element>.fontFamily` | 元素字体族 | 字符串 / CSS font-family | 非空字符串 | 正文类由 `font.familySans` 派生；`inlineCode`、`codeBlock` 由 `font.familyMono` 派生 |

| `markdown.<element>.color` | 文字颜色 | HEX 颜色 | `#RRGGBB` | 通常派生自 `ui.foreground`；链接取 `ui.primary`，引用取 `ui.mutedForeground`，行内代码取 `ui.destructive` |

纵向节奏以 4px 为基本阶梯。标题前距表达“新章节开始”，因此显著大于标题后距；标题后距表达标题与所属内容的紧密关系。连续标题不重复套用章节前距，实际间距取 `headingStackSpacing` 与后一级标题 `marginBottom` 的较大值，因此 H1–H6 仍保留递减层级。文档首块的上间距和末块的下间距固定清零。

普通块之间保留 CSS 标准外边距折叠，因此相邻块的实际距离是前块 `marginBottom` 与后块 `marginTop` 的较大值，而不是两者相加。列表的同级条目、进入下一层级和返回上一层级统一由对应列表的 `itemSpacing` 控制，层级结束处不会再次叠加外边距。所有纵向间距会跟随 Markdown 文字缩放；横向阅读内边距、列表标记栏、标记间距和嵌套缩进保持物理像素值。

`emphasis` 与 `strong` 没有专属字段，差异完全由各自的六个文字字段表达。主题面板、CSS 编译和序列化只消费以上当前路径。

#### v4–v7 兼容迁移

旧主题仍可作为输入隐式迁移，优先级固定为：当前元素字段 > v7 `markdown.typography` > `typography.markdown` > 更早的 `markdown.font` 与元素内排版字段 > 当前默认值。旧 `markdown.quote` 迁移为 `markdown.blockquote`；旧列表外观复制到 `list.unordered`、`list.ordered` 和 `list.task`；旧强调外观复制到 `emphasis` 与 `strong`，其中 `strongWeight` 只写入 `strong.fontWeight`。归一化和再次保存后只保留当前结构，不再输出兼容字段。

## 动效

动效 token 是应用级运动语言，服务浮动控件、覆盖面板、工作视图切换、终端面板和无限画布视觉补间。GSAP 只能消费这些 token，不能在组件中写死时长、缓动、距离或动画上限。

| Token | 含义 | 当前字段 / CSS 映射 |
| --- | --- | --- |
| `motion.duration.fast` | 按钮、hover、短反馈时长 | `motion.duration.fast` / `--motion-duration-fast` |

## 实现边界

- `EditorTheme` 当前版本为 v11，覆盖颜色、字体、完整 Markdown 元素外观、尺寸、圆角、线宽、虚线、图标、画布交互、终端、ANSI 16 色、分组、连线标签、网格和诊断 token。
- `compileEditorTheme()` 是运行时唯一编译入口，统一产出 CSS 变量、`CanvasVisualTokens`、几何 token、网格 spec、Mermaid themeVariables、xterm theme 和主题诊断。
- 文件级主题写入 `canvas-layout.theme`，旧文件无该字段时继续使用本地应用偏好。
- 浮动主题设置面板按资源树式分类开放应用、Markdown、画布、终端等 Token；Markdown 再按基础、标题、行内语义和块级语义分组，每个元素可以独立调整与重置。
- Mermaid 渲染视图只能通过 `themeVariables` 接收部分 token，无法 1:1 接收所有画布几何和交互 token。

# 交互状态层设计约束

本文档定义无限画布的交互状态层级。后续新增画布行为时，必须先进入这里定义的状态和事件分发模型，避免在 Konva 组件事件里分散提交业务行为。

## 状态层级

输入事件按以下顺序解释：

1. 输入归属：文本框、内联编辑、菜单、画布、渲染视图。
2. 工作视图：只有无限画布处理画布编辑交互，渲染视图不复用画布状态。
3. 瞬时状态：拖拽、框选、平移、连线、重连、编辑优先于持久工具。
4. 持久工具：只保留 `select` 和 `connect`。
5. 命中目标：空白、节点、节点锚点、连线、连线标签、连线端点。
6. 命令执行：选择、清空选择、移动、框选、新增节点、创建连线、重连、文本编辑。

## 持久工具

- `select` 是默认工具，负责选择、框选、拖拽、空白双击新增、文本编辑。
- `connect` 负责从节点锚点拖出连线。
- `pan` 不是持久工具。按住 Space 或鼠标中键时进入临时平移状态，松开或抬起鼠标后退出。

## 瞬时状态

- `idle`：没有正在进行的画布交互。
- `pendingBlankPointer`：空白按下后，等待判断是点击还是框选。
- `pendingNodePointer`：节点按下后，等待判断是点击还是拖拽。
- `marqueeSelecting`：空白拖拽框选。
- `draggingNodes`：拖拽一个或多个节点，允许对齐吸附和辅助线。
- `panning`：临时平移画布，不改变选择，不写入历史。
- `connectingEdge`：从节点锚点拖出连线。
- `retargetingEdge`：拖拽连线端点重连。
- `editingNodeText`：编辑节点文本。
- `editingEdgeLabel`：编辑连线文本。

## 空白双击新增

保留空白双击新增节点，但必须通过空白点击意图保护：

- 两次点击都必须命中空白。
- 两次点击之间不能发生选择变化、节点/边点击、拖拽、框选、平移、连线、重连或文本编辑。
- 已选中内容时，第一次空白点击只清空选择，并消费该点击。
- 节点点击后再点击空白，不能合成新增节点。

## 鼠标指针

鼠标指针样式必须由持久工具和瞬时状态统一派生：

- `idle + select`：默认指针。
- `idle + connect` 或连线/重连中：十字指针。
- Space 可平移但尚未拖动：抓取指针。
- 正在平移或拖拽节点：抓取中指针。
- 文本编辑中：文本指针。

## 实现约束

- `canvas-interaction.ts` 是画布交互状态机的核心约束层。
- Konva 组件只负责把原始事件转换为标准输入，并执行状态机返回的命令或副作用。
- 历史记录只在明确提交边界写入；拖拽中的移动只做 draft 更新。
- 对齐辅助线、连线草稿、框选矩形是瞬时状态的可视副作用，不应写入 Mermaid 文件。

## 节点几何层

- `node-geometry.ts` 是节点卡片几何的唯一来源。
- 节点 frame、文本框、连接锚点、边路由矩形、对齐矩形、命中测试都必须从同一份 `NodeGeometry` 派生。
- 节点的 Konva 结构必须是一个可拖拽 Group 包含 Rect、Text 和四个连接点；连接点使用节点局部坐标，不能作为节点 Group 的兄弟元素单独定位。
- 连接点只表示连线交互入口，不参与对齐吸附；对齐吸附使用节点 frame 或多选 selection bounds。
- 拖拽吸附算出 snapped position 后，必须同步更新正在拖拽的 Konva Group position 和 graph draft，避免视觉位置与数据位置分裂。

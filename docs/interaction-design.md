# 交互状态层设计约束

本文档定义无限画布的交互状态层级。后续新增画布行为时，必须先进入这里定义的状态和事件分发模型，避免在 Konva 组件事件里分散提交业务行为。

## 状态层级

输入事件按以下顺序解释：

1. 输入归属：文本框、内联编辑、菜单、画布、渲染视图。
2. 工作视图：只有无限画布处理画布编辑交互，渲染视图不复用画布状态。
3. 瞬时状态：拖拽、框选、平移、连线、重连、编辑优先于持久工具。
4. 持久工具：只保留 `select` 和 `connect`。
5. 命中目标：空白、节点、节点锚点、连线、连线标签、连线端点。
6. 命令执行：选择、清空选择、移动、框选、新增节点、创建连线、重连、文本编辑。

## 持久工具

- `select` 是默认工具，负责选择、框选、拖拽、空白双击新增、文本编辑。
- `select` 模式下，节点 hover 或 selected 时显示连接点；从连接点拖拽可以创建连线。
- `connect` 是整节点连线模式，不显示连接点；从节点主体按下拖到另一个节点即可创建连线。
- `pan` 不是持久工具。按住 Space 或鼠标中键时进入临时平移状态，松开或抬起鼠标后退出。

## 瞬时状态

- `idle`：没有正在进行的画布交互。
- `pendingBlankPointer`：空白按下后，等待判断是点击还是框选。
- `pendingNodePointer`：节点按下后，等待判断是点击还是拖拽。
- `marqueeSelecting`：空白拖拽框选。
- `draggingNodes`：拖拽一个或多个节点，允许对齐吸附和辅助线。
- `panning`：临时平移画布，不改变选择，不写入历史。
- `connectingEdge`：从节点锚点拖出连线。
- `retargetingEdge`：拖拽连线端点重连。
- `editingNodeText`：编辑节点文本。
- `editingEdgeLabel`：编辑连线文本。

## 空白双击新增

保留空白双击新增节点，但必须通过空白点击意图保护：

- 两次点击都必须命中空白。
- 两次点击之间不能发生选择变化、节点/边点击、拖拽、框选、平移、连线、重连或文本编辑。
- 已选中内容时，第一次空白点击只清空选择，并消费该点击。
- 节点点击后再点击空白，不能合成新增节点。

## 鼠标指针

鼠标指针样式必须由持久工具和瞬时状态统一派生：

- `idle + select`：默认指针。
- `idle + connect` 或连线/重连中：十字指针。
- `idle + select` 悬停在节点连接点或连线端点手柄上：十字指针。
- Space 可平移但尚未拖动：抓取指针。
- 正在平移或拖拽节点：抓取中指针。
- 文本编辑中：文本指针。

## 触控板和滚轮导航

无限画布和渲染视图必须把 MacBook 触控板作为一等输入，并共享同一套导航语义：

- 无修饰键双指滚动平移画布，不缩放。
- Shift + 滚动在没有横向滚动量时转为水平平移。
- Cmd/Ctrl + 滚动缩放画布，缩放中心为当前指针位置。
- Safari `gesture*` 捏合事件缩放画布，缩放中心为手势开始位置。
- 触控板、滚轮和捏合只改变 viewport，不写入历史。
- 高频导航必须先命令式更新视觉 viewport，再低频提交 React viewport 状态，避免触控板事件被 React 重渲染节奏限制。
- 拖拽、框选、连线、重连、内联编辑等瞬时状态中忽略滚轮导航。
- 画布容器必须阻止浏览器接管触摸平移/缩放，源码面板、属性面板和渲染视图保持各自原有滚动行为。

## 实现约束

- `canvas-interaction.ts` 是画布交互状态机的核心约束层。
- `canvas-viewport-navigation.ts` 是滚轮、触控板和手势到 viewport 变更的唯一转换层。
- Konva 组件只负责把原始事件转换为标准输入，并执行状态机返回的命令或副作用。
- 历史记录只在明确提交边界写入；拖拽中的移动只做 draft 更新。
- 对齐辅助线、连线草稿、框选矩形是瞬时状态的可视副作用，不应写入 Mermaid 文件。

## 命中目标层

- `canvas-hit-target.ts` 是 Konva 图形命中到业务目标的唯一转换层。
- 节点、连接点、连线、连线标签和连线端点必须在 Konva 图形上写入稳定的业务 hit id。
- 事件入口必须先解析为 `HitTarget`，再交给 `canvas-interaction.ts`；不能在具体 Rect、Text、Circle、Arrow handler 中直接写选择、编辑、连线业务判断。
- 子图形命中必须向父级查找最近业务目标，例如节点文字命中节点、标签文字命中连线标签。

## 命令执行层

- `canvas-interaction.ts` 只返回 `CanvasInteractionCommand`，不能直接修改 React state、graph、selection、viewport 或历史记录。
- Konva 组件只在命令执行边界调用 `onSelectionChange`、`onGraphDraft`、`onGraphCommit`、`onViewportChange`、`onCaptureHistory` 和内联编辑状态。
- 空白双击、新增节点、节点/连线选择、文本编辑、连线创建、端点重连、框选选择都必须经过命令执行层。
- 鼠标右键、中键和 Space 临时平移优先于任何业务命中目标。

## 连接预览层

- `connection-preview.ts` 是连接创建和端点重连的目标解析层。
- Konva 组件必须把当前拖拽状态、鼠标世界坐标和节点几何传入预览层，由预览层返回有效性、目标节点、无效节点和路由目标。
- 创建连线时，非源节点是有效目标；源节点和空白是无效目标。
- 端点重连时，新节点是有效目标；当前同侧端点节点和空白是无效目标；拖到另一端节点允许形成自环。
- 有效目标节点必须高亮，无效节点必须使用弱化的无效状态；无效拖拽松手静默取消，不弹提示。
- 连接创建、端点重连和最终完成线必须使用同一套路由目标解析，避免预览和提交结果不一致。

## 节点几何层

- `node-geometry.ts` 是节点卡片几何的唯一来源。
- 节点 frame、文本框、连接锚点、边路由矩形、对齐矩形、命中测试都必须从同一份 `NodeGeometry` 派生。
- 节点的 Konva 结构必须是一个可拖拽 Group 包含 Rect、Text 和四个连接点；连接点使用节点局部坐标，不能作为节点 Group 的兄弟元素单独定位。
- 连接点只表示连线交互入口，不参与对齐吸附；对齐吸附使用节点 frame 或多选 selection bounds。
- 拖拽吸附算出 snapped position 后，必须同步更新正在拖拽的 Konva Group position 和 graph draft，避免视觉位置与数据位置分裂。

## 视觉反馈层

- `canvas-visual-state.ts` 是画布视觉状态的唯一来源，负责节点、连线、连接点、框选、连线草稿、对齐辅助线和端点手柄的颜色、描边、显隐与优先级。
- Konva 组件只传入 `mode`、`selection`、hover 目标、`InteractionState` 和内联编辑目标，不直接写 hover、selected、dragging、editing 的视觉分支。
- 节点视觉优先级为 `editing > dragging > connectionInvalid/connectionTarget > selected > hovered > normal`。
- 连接点显隐由视觉层统一判断；连接点只在 `select` 模式下作为精确连线入口，不在 `connect` 模式显示，也不能参与节点几何、边路由或对齐吸附计算。
- 连线端点圆点只表示端点重连 handle，不参与选择，也不能作为可独立移动的画布对象；其视觉状态必须区分 normal、hovered 和 active。
- 草稿线和重连预览线必须区分 valid 和 invalid 状态；invalid 只弱化视觉，不改变提交规则。
- 实线、虚线、粗线等连线语义样式必须保留在视觉层，不能散落到渲染组件里。

## 连线路由层

- `edge-geometry.ts` 是无限画布连线路径的唯一来源。
- 完成连线和拖拽中的草稿连线必须使用同一套全局 `edgeRouting` 路由策略。
- 草稿线拖到目标节点上时，路径必须与松开后生成的完成线一致；拖到空白处时，终点跟随鼠标但仍保持同类直线或曲线风格。
